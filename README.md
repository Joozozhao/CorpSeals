# CorpSeals

CorpSeals 是一款面向本地办公场景的 PDF 电子签章工具，支持主体管理、印章扫描件生成、PDF 盖章、骑缝章、主体资质文件管理和本地数据持久化。项目基于 React、Vite、Electron 和 pdf-lib 构建，可作为网页工具运行，也可以打包成 macOS 桌面应用。

这个项目的目标不是做一个重型合同系统，而是提供一个轻量、可复用、离线优先的签章工作台：用户可以在本机维护多个签章主体，上传清晰的印章扫描件，生成透明 PNG 印章，并将印章拖拽放置到 PDF 页面上完成导出。

## 功能特性

- 多主体管理：为不同公司或签章主体维护独立印章资料。
- 多类型印章：支持公章、合同章、财务章、法人章。
- 扫描件生成印章：上传清晰红色扫描件后，自动去白底、裁剪多余边缘并生成透明 PNG。
- 固定印章尺寸：公章按 40mm x 40mm，法人章按 18mm x 18mm 处理。
- PDF 盖章：支持批量盖章、指定页盖章，并可直接在页面上拖拽调整位置。
- 单页位置微调：批量盖章时，可为某一页单独设置印章位置。
- 骑缝章：支持按页面拆分骑缝章，并通过鼠标拖拽调整位置。
- 主体资质管理：可为主体上传营业执照、授权书等资质文件。
- 资质下载水印：资质文件下载时可添加公章和用途水印，水印自动包含下载时间戳。
- 本地数据保存：桌面应用模式下，主体、印章和资质数据保存到当前用户的应用数据目录，不写入应用安装包。
- 本地登录保护：默认用户名 `joozo`，密码 `7777`，适合本机私有使用场景。

## 本地运行

安装依赖：

```bash
npm install
```

启动 Web 开发服务：

```bash
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173/
```

如果需要启动桌面应用：

```bash
npm run electron
```

## 构建

构建前端资源：

```bash
npm run build
```

打包 macOS DMG：

```bash
npm run dist:mac
```

打包 Windows 安装包和免安装包：

```bash
npm run dist:win
```

打包产物默认输出到 `release/` 目录。

Windows 打包配置会生成 NSIS 安装包和 portable 免安装版。若在 macOS 上交叉打包 Windows 应用，可能需要额外的本机打包环境支持；最稳妥的方式是在 Windows 机器或 Windows CI 环境中执行 `npm run dist:win`。

## 数据存储

在普通浏览器开发模式下，数据会保存在浏览器本地数据库中，方便开发调试。

在 Electron 桌面应用中，数据会写入当前用户的应用数据目录，例如 macOS 下通常类似：

```text
~/Library/Application Support/CorpSeals/data/workspace.json
```

这样可以保持应用安装包本身干净可复用，不把某个公司的印章和资质数据打进 APP 内部。

## 使用说明

1. 登录工作台。
2. 添加签章主体。
3. 为主体上传公章、合同章、财务章或法人章扫描件。
4. 上传需要盖章的 PDF。
5. 选择批量、指定页或骑缝章模式。
6. 在 PDF 页面上拖拽调整印章位置。
7. 导出已盖章 PDF。

## 适用场景

- 本地 PDF 文件快速盖章。
- 多主体印章资料集中管理。
- 小团队内部的合同、资质、授权文件处理。
- 离线优先的法务或行政办公工具。
- Electron 桌面端签章工具二次开发。

## 注意事项

- 请仅上传和使用你有权使用的印章图片。
- 本工具不会验证印章、签署行为或文件内容的法律效力。
- 默认账号密码仅用于本地轻量保护，不应视为企业级安全认证。
- 如需多人协作、权限控制、审计日志或电子签名合规能力，需要接入更完整的后端和认证体系。
- 开源发布前请确认示例数据、印章图片和构建产物不包含敏感信息。
